GreenCare Codebase Exploration - Summary Report

PROJECT OVERVIEW
================

Project Type: Full-stack plant care management system with AI
Architecture: Turbo monorepo with Express backend and React frontend
Database: PostgreSQL with Prisma ORM

TECH STACK
==========

Frontend:
- React 18.3.1, TypeScript, Vite 5.4.1
- React Router 7.6.3, Axios 1.10.0
- Tailwind CSS 4.1.11 (dark mode support)
- HeroUI 2.8.1 (component library)
- Chart.js for data visualization
- Icons: Lucide (primary), Heroicons, React Icons, HugeIcons
- Sonner for notifications

Backend:
- Express 4.21.0 with TypeScript
- Prisma 5.19.1 ORM for PostgreSQL
- Passport.js + Google OAuth 2.0
- JWT authentication with bcrypt
- Multer for file uploads
- AWS S3 for cloud storage

AI FEATURES IMPLEMENTED
=======================

Backend Services (Google Gemini 2.5 Pro):
1. Water pH Recommendation
2. Comprehensive Care Recommendations (watering, light, temperature, fertilization, seasonal)
3. Plant Disease Diagnosis (symptoms, causes, treatment, severity)
4. Growth Prediction (height, width, milestones, timeline)
5. Companion Plant Suggestions (4-6 recommendations)
6. Environmental Optimization (light, temperature, humidity, air circulation)
7. Plant Care Chat Assistant (conversational AI)

API Endpoints (/api/v1/ai/):
- POST /care-recommendations
- POST /disease-diagnosis
- POST /growth-prediction
- POST /companion-plants
- POST /environmental-optimization
- POST /chat

Frontend Components:
1. AIAnalyticsDashboard - Performance metrics and insights
2. AICareDashboard - Central management hub
3. AICareScheduler - AI-generated schedules with weather awareness
4. AINutritionAnalyzer - Soil analysis and fertilizer recommendations
5. AIPropagationGuide - Propagation instructions
6. AISeasonalPlanner - Weather-aware seasonal planning
7. AIStressDetector - Stress analysis (environmental, nutritional, pest, disease)

ROUTING STRUCTURE
=================

Public Routes:
- / (Landing page with hero, benefits, story, community, FAQ)
- /auth/signup
- /auth/signin
- /auth/google/success

Protected Routes (Require JWT):
- /dashboard (Main hub - plants, tasks, feed)
- /plants (Plant collection)
- /care (Care management)
- /find-plant (Plant discovery/identification)
- /community (Social feed)
- /ai-features (AI showcase)
- /advanced-ai-features (Advanced tools)

AUTHENTICATION
===============

Methods:
- JWT (jsonwebtoken)
- Google OAuth 2.0 (Passport.js)
- Password hashing (bcrypt)

Implementation:
- Tokens stored in localStorage
- Axios interceptor adds token to requests
- 401 responses trigger re-authentication
- Session management with express-session

FILE STORAGE & APIS
===================

Cloud Storage: AWS S3 (apps/api/src/s3.ts)

External APIs:
1. Google Gemini 2.5 Pro - AI recommendations
2. Plant.id API - Plant identification from images
3. Google OAuth 2.0 - Social authentication
4. AWS S3 SDK - Cloud storage
5. Weather APIs - Environmental data (via useWeather hook)

DATABASE MODELS
===============

Using Prisma with PostgreSQL:
- User (email, password, profile)
- Plant (species, location, health status)
- PlantSpecies (care requirements database)
- CareSchedule, CareTask, CareLog (task management)
- HealthLog (health tracking)
- CommunityPost, PostComment, PostLike (social)
- PlantImage (multiple images per plant)
- PlantIdentification (API results storage)
